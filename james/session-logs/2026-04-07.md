**Decisions:**
- Add download fields to existing `viral_videos` table (via idempotent ALTER) rather than creating a parallel table with duplicated metadata — single source of truth
- English filter operates on **title only**, not title+description. Descriptions are keyword-stuffed with English SEO hashtags even on non-English videos, which drowned out the Devanagari/CJK title signal and was accepting Hindi videos incorrectly
- Unicode script detection via explicit codepoint ranges beats `.isalpha()` which skips combining marks like Devanagari vowel signs (was undercounting by ~50%)
- `skipped` is its own download status (intentional/permanent), separate from `failed` (retryable in principle)
- Status enum on `viral_videos.download_status` is a plain string column — no `Literal[...]` type to match existing codebase convention
- Search query = market title verbatim. No LLM entity extraction yet. Cheapest path first
- Any market in scope (not just mention markets / entertainment) — "just start with any topic"
- No scheduler hookup or production-worker integration this session — CLI only
- Stitching multiple clips into derivative Rekko videos is the accepted TOS posture; no re-uploading raw clips
